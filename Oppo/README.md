# 项目

## 项目准备

- [ ] 项目流程图

- [ ] 项目总访问量：估算CDN费用 `总量 * 页面大小`

- [ ] 项目峰值QPS `QPS = req/sec = 请求数/秒`：决定服务器数（单台qps 低于500 巅峰过1000 就得备3台）。对于读>800QPS以上，写>300QPS（基于1台AP（8核心/16G的情况下），其他服务器任意）

- [ ] 项目后端接口，所有页面都要部署埋点统计，并保证数据的准确性

- [ ] 前端优化：

      1、减少http请求，减小cookie体积

      2、优化图片大小并使用分屏加载

      3、js按需加载

      4、压缩html、css、js代码

      5、使用cdn和gzip压缩，并开启缓存
